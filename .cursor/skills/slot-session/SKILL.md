# 実践セッション相談

## いつ使うか

- 実践中の状況共有（G数、挙動、示唆、差枚）
- 「この挙動どう読む？」「期待値ある？」「ヤメ時？」など

## 手順

1. 不足情報を最短で確認する（分かっている項目は聞かない）
   - 機種名
   - 店舗（分かれば）
   - 現在G / 直近ボーナス・CZ
   - 気になる挙動・示唆
   - 目的（設定狙い / ゾーン消化 / イベント拾い）
2. `knowledge/machines/<機種>.md` を読む。無ければ作成候補を出す
3. 店舗があれば `knowledge/stores/` と `knowledge/events/` も参照。**未指定なら README のよく行く店を優先**
4. `playbooks/zone-check.md` と `playbooks/setting-inference.md` に沿って整理
5. 結論を短く返す（結論 → 根拠 → 次の確認）

## 出力テンプレ

```markdown
## 結論
（1行）

## 読み
- ...

## 期待値・設定
- 確定: ...
- 有力: ...
- 可能性: ...
- 不明: ...

## 次の一手
1. ...
```

## 記録

ユーザーが同意したら `sessions/` にログを作成・追記する。
テンプレは `sessions/_template.md`。

## 終了手順（セッションが終わったら）

実践終了の共有（ヤメ・閉店・撤退）を受けたら、次の3点をセットで行う:

1. **収支台帳**: 投資・回収・収支を確認し `sessions/ledger.md` に1行追記（不明な項目は「不明」のまま）
2. **店舗観測ログ**: 店舗が分かる実践なら `knowledge/stores/<店舗>.md` の観測ログに1行追記
   （日付・総差枚や自台の結果・強かった末尾など、その日に分かった範囲で）
3. **機種知見**: 再現性のありそうな挙動・示唆の知見は `knowledge/machines/<機種>.md` へ昇格

当日 `knowledge/events/` に予想ファイルがある日は、翌日レビュー（`playbooks/x-event-analysis.md` セクション5）の対象になることも一言添える。
