# 店舗・イベント調査

## 手順

1. `knowledge/stores/README.md` で**よく行く店・たまに行く店**の優先度を確認
2. `knowledge/stores/` と `knowledge/events/` を確認
3. プレイブックを参照:
   - `playbooks/store-scouting.md` … 全体方針
   - `playbooks/information-sources.md` … 情報源
   - `playbooks/x-event-analysis.md` … X告知×過去結果
   - `playbooks/seat-number-analysis.md` … 台番・末尾
4. 店舗の **X告知** と **みんレポ等の過去結果** を突合（当日情報は Web 検索）
5. **カタログの X 情報アカウント（東スロ等）は当日予想の主根拠として強く反映**する
6. X情報が公式・設置・日付などと矛盾し一般的に誤りなら、**指摘して除外／減点**する
7. 台番・末尾の癖は `knowledge/stores/` の観測ログを優先、なければデータサイトから考察
8. 推測は「確定 / 有力 / 可能性 / 不明」に分ける（情報垢推しは最大「有力」、確定は店舗公式のみ）
9. 実践向けに「今日の仮説」と「観測ポイント」を短く返す

## 店舗提案のルール

- **優先度1（よく行く店）から提案を組み立てる**。店名リストは `knowledge/stores/README.md` が一次ソース
- 旧イベ・データが別店に有利でも、訪問頻度が低い店を最優先にしない
- 優先度2（たまに行く店）はカレンダー一致時のみ候補に上げる
- X情報垢が遠征店を推していても、よく行く店より上にしない（遠征はユーザーOK時に併記）

## 出力テンプレ

```markdown
## 店舗仮説
- ...

## イベント読み（X・旧イベ）
- 確定: ...（店舗公式のみ）
- 有力: ...（X情報垢・直近データを主根拠にしてよい）
- 指摘: ...（Xが一般的に誤りならここに書く）

## 台番・末尾
- 有力: ...
- 可能性: ...

## 狙い目
1. ...

## 観測ポイント
- ...

## 注意
- 未確認情報はここに明示
```

## 記録

- 新しい傾向 → `knowledge/stores/<店舗>.md`
- 特定日の告知と結果 → `knowledge/events/YYYY-MM-DD-<店舗>.md`
- テンプレは `knowledge/stores/_template.md` `knowledge/events/_template.md`
