# 情報源カタログ（スロット実践調査）

店舗・イベント・台番の傾向を調べるときの**参照先一覧**。  
「公式に確かなこと」と「第三者データ・仮説」は必ず分けて記録する。

## 優先順位

### 事実の確定（信頼の階層）

1. 店舗公式（X / LINE / HP / 店内告知）… 日付・公約・開店の**確定**
2. このリポジトリの `knowledge/stores/` `knowledge/events/`（過去に突合済みの傾向）
3. 台データ公開サイト（みんレポ、DMMぱちタウン、P-WORLD）… 結果の検証
4. 取材結果サイト（一撃、rx7038、モグ玄荒修行 など）
5. 口コミ・まとめサイト（みんパチ、ブログ）→ **弱い仮説**

### 当日予想への影響度（新しさ重視）

**当日・翌日の立ち回り予想では、カタログ記載の X 情報アカウントを強く反映する。**  
（リアルタイム性が高く、旧イベ固定や古い口コミより「今」の材料になりやすい、という運用方針）

| 順位 | ソース | 予想への重み |
|------|--------|--------------|
| 1 | 店舗公式Xの当日告知 | 最大（確定事実） |
| 2 | X情報アカウント（東スロ・関東情報局など・下記） | **強い有力**（予想の主根拠にしてよい） |
| 3 | みんレポ等の直近同条件結果 | 強い検証・修正材料 |
| 4 | knowledge の過去傾向・旧イベ | ベースライン |
| 5 | 口コミ・古いまとめ | 弱い補助 |

矛盾時の裁き:

1. **店舗公式と矛盾** → 公式を取り、情報垢側を「誤り候補」として明示指摘
2. **複数の有力X垢が同方向** → 予想の主根拠として採用（有力）
3. **X垢同士が割れる** → よく行く店優先＋みんレポで切る。割れている旨を書く
4. **一般常識・スペック・設置と明らかに矛盾**（例: 未設置機種の全台、閉店店の推し）→ **誤りとして指摘**し、予想から落とす
5. **投稿日時が古い／対象日ズレ** → 重みを落とすか捨てる

## X 情報アカウント（第三者まとめ）

店舗公式より先に「明日どこが熱いか」の地図を作り、**当日予想の主根拠に据える**。  
最新性が高いため影響度は大きいが、**確定（事実）には昇格させない**。確定は店舗公式のみ。

### 確度の付け方（X垢起点）

| X側の状態 | 予想での確度 | 応答での書き方 |
|-----------|--------------|----------------|
| 有力カタログ垢が注目＋根拠（取材名・差枚・示唆）あり | **有力**（主根拠可） | 「X根拠: …のため優先」 |
| 複数カタログ垢が同方向 | **有力〜強有力** | 両方を根拠に併記 |
| 単発の推し・根拠薄い | **可能性** | 補助扱い |
| 公式・設置・日付と矛盾 | **誤り候補** | **指摘して捨てる／大幅減点** |
| ハンドル停止・古い投稿 | **不明** | 使わない |

扱いルール:

| 扱い | 内容 |
|------|------|
| する | 東スロ等の当日推し・取材一覧を**予想の主材料**にする |
| する | 明らかに誤っている情報は**黙殺せず指摘**する |
| しないこと | 情報垢の推しで、よく行く店ルールを無視して別店を最優先にする |
| しないこと | X推しを「確定」と書く（公式以外は確定にしない） |
| 突合 | よく行く店に刺さる話・誤り指摘は `knowledge/events/` へ残す |
| ハンドル | 凍結・改名あり得る。投稿が止まっていたら「不明」扱いに落とす |

### 誤りとみなす典型（指摘テンプレ）

次に当てはまる場合は、結論で一言指摘してから別案を出す:

- 対象日のズレ（「明日」投稿なのに日付が違う）
- 店舗公式告知と内容が食い違う
- その店に無い機種・無い台数帯の公約
- 閉店・休業・移設後の旧店名での推し
- 同系統アカウント間で事実（店名・日付・取材有無）が一致しない
- 差枚ランキングの日付が明示されず古い可能性がある

指摘例:

```
指摘: @xxxx の推しは未設置機種の全台になっており一般的に誤り候補。予想から除外。
代替（有力）: 店舗公式の〜告知＋@touslot の取材一覧に沿って優先度1店を軸に組む。
```

### 使い分け（実務）

```
前夜〜朝:
  1. @touslot / 東スロブログ … 明日の取材・注目（予想の主材料）
  2. @p_info_kanto / @karasuro_7 / @slot_brain … 神奈川差し込み（同方向なら強く採用）
  3. よく行く店の店舗公式X … 確定告知・矛盾チェック（誤り指摘の基準）
  4. みんレポ等 … 直近同条件で検証・修正
```

店舗提案に落とすとき:

1. X情報アカウントの注目を**予想の主根拠**として書き、確度は有力以上なら順序を前にする
2. ただし注目店 ⊂ `knowledge/stores/README.md` の優先度1〜2 を第一候補とする（遠征はユーザーOK時）
3. 優先度3や未登録店へのX推しは「遠征候補」として併記し、よく行く店より上にしない
4. 「公式告知あり」のみ確定。「X推し」は有力（誤り指摘がない限り予想に強く効かせる）
5. 誤りの可能性があるX情報は、結論の直下で指摘してから別案を出す

### データサイトの見方（最低限）

1. **総差枚・平均差枚・勝率** … 店のその日の「波」
2. **平均G数** … 後ヅモでは1500以上のプラス台を優先
3. **機種別** … 並び・全台系の候補
4. **バラエティ（1台設置）** … 散らし拾い
5. **末尾別** … 仕掛けの数字仮説（後述プレイブック参照）

## 記録のルール

新しい情報は以下に残す:

| 内容 | 保存先 |
|------|--------|
| 店舗の恒常傾向 | `knowledge/stores/<店舗>.md` |
| 特定日のイベント・結果 | `knowledge/events/YYYY-MM-DD-<店舗>.md` |
| 実践ログ | `sessions/YYYY-MM-DD-<機種or店舗>.md` |

各ファイルで必ず分ける:

- **確定** … 公式告知・公式サイトの事実
- **有力** … 複数日のデータで再現した傾向
- **可能性** … 1〜2回だけの偏り、未検証の仮説
- **不明** … 未確認
