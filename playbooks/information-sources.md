# 情報源カタログ（スロット実践調査）

店舗・イベント・台番の傾向を調べるときの**参照先一覧**。  
「公式に確かなこと」と「第三者データ・仮説」は必ず分けて記録する。

## 優先順位

1. このリポジトリの `knowledge/stores/` `knowledge/events/`
2. 店舗公式（X / LINE / HP / 店内告知）
3. 台データ公開サイト（みんレポ、DMMぱちタウン、P-WORLD）
4. 取材結果サイト（一撃、rx7038、モグ玄荒修行 など）
5. 口コミ・まとめサイト（みんパチ、ブログ）→ **仮説扱い**

## 店舗公式・グループ

| 種別 | 用途 | 例（アビバ綱島） |
|------|------|------------------|
| X（旧Twitter） | **当日・翌日のイベント告知**、機種入替、周年 | 店舗アカウント、グループアカウント |
| LINE | 抽選時間変更、特定日の注意 | 店舗LINE（みんパチにURL記載あり） |
| 店舗HP / P-WORLD | 台数、換金、入場、機種リスト | P-WORLD店舗ページ |
| DMMぱちタウン | 台データ、来店レポート、新台 | 店舗ページ `/jackpot` |
| 店内告知 | 取材日・機種指定は最優先で信頼 | 現地確認 |

### X を見るときのチェック項目

- [ ] 告知日時と対象日（「明日」「○/○」のズレに注意）
- [ ] 対象機種・台数・「全台」「並び」「末尾」などのキーワード
- [ ] 画像付き告知（機種名・台番レンジが書いてあることが多い）
- [ ] リポスト・引用（グループ合同イベントの有無）
- [ ] 過去の同曜日・同イベント日の告知パターンとの比較

## 台データ・差枚

| サイト | URL例 | 強み | 弱み |
|--------|-------|------|------|
| **みんレポ** | min-repo.com | 機種別・**末尾別**・バラエティ単品、過去一覧 | 全店カバーではない、ATのG数誤差あり |
| **DMMぱちタウン** | p-town.dmm.com | リアルタイム寄り、フロアマップ | 店舗によって更新頻度差 |
| **P-WORLD** | p-world.co.jp | 機種構成・新台・基本情報 | 差枚は弱い |
| **モグ玄荒修行** | mogugen.com | **台番別BB/RB/ART**、末尾集計、連続プラス | 全店ではない |

### データサイトの見方（最低限）

1. **総差枚・平均差枚・勝率** … 店のその日の「波」
2. **平均G数** … 後ヅモでは1500以上のプラス台を優先
3. **機種別** … 並び・全台系の候補
4. **バラエティ（1台設置）** … 散らし拾い
5. **末尾別** … 仕掛けの数字仮説（後述プレイブック参照）

## 記録のルール

新しい情報は以下に残す:

| 内容 | 保存先 |
|------|--------|
| 店舗の恒常傾向 | `knowledge/stores/<店舗>.md` |
| 特定日のイベント・結果 | `knowledge/events/YYYY-MM-DD-<店舗>.md` |
| 実践ログ | `sessions/YYYY-MM-DD-<機種or店舗>.md` |

各ファイルで必ず分ける:

- **確定** … 公式告知・公式サイトの事実
- **有力** … 複数日のデータで再現した傾向
- **可能性** … 1〜2回だけの偏り、未検証の仮説
- **不明** … 未確認
