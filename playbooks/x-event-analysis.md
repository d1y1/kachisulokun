# X告知・イベント結果の考察プレイブック

店舗の X 告知と、過去のイベント結果を突き合わせて**当日の仮説**を作る手順。

情報アカウント一覧（東スロ・関東パチスロ情報局など）は `playbooks/information-sources.md` の「X 情報アカウント」を参照。

## 0. 予測への重み（運用方針）

- **最新の X（店舗公式＋カタログ情報垢）は、当日予想の主根拠として強く効かせる**
- ただし一般的に誤りと判断できる場合は、**指摘して除外／減点**する（黙って採用しない）
- 「確定」は店舗公式のみ。情報垢は最大でも「有力」

誤り判定の最低ラインは `information-sources.md` の「誤りとみなす典型」に従う。

## 1. 告知の読み取り

### キーワード辞書

| 告知文言 | 読み方（一般的） | 確認すること |
|----------|------------------|--------------|
| 全台系 / 全台 | その機種の全台に高設定の可能性 | 機種名・設置台数 |
| 並び / ○台並び | 連続台番に設定塊 | 何台並びか、機種 |
| 末尾 / ○のつく台 | 台番の一の位・十の位で仕掛け | 対象数字（1,7,8 など） |
| ゾロ目 | 下二桁が同じ（11,22…） | 月日ゾロ目と合わせて見る |
| 1/2 / 半数 | 設置の半分に高設定 | 機種・エリア |
| 取材 / ○○来店 | 特定機種・特定列が対象になりやすい | 過去の同取材の結果 |
| 時差開放 / ○時～ | 午後から特定機種が狙い目 | 解放時刻・対象機種 |
| ジャグラー系 | Aタイプ中心の仕掛け | AT狙いなら優先度下げ |

### 告知→結果の突合手順

1. X で対象日の告知を保存（スクショ or 文言メモ）
2. 翌日以降、みんレポ等で**総差枚・機種別・末尾別**を確認
3. 告知に書いてあった機種が本当に強かったか照合
4. `knowledge/events/` に「告知内容」「結果」「一致/不一致」を記録
5. 2回以上再現したら `knowledge/stores/` の傾向に昇格

## 2. 旧イベント日の使い方

みんパチ等の「旧イベント日」は**過去に強かった日**の目安。  
**毎回当たる保証ではない**。

### 考察の型

```
旧イベ日 + 公式の追加公約（みんパチ口コミ等）
  → 例: 1のつく日 + アビバ全店9時開店
  → 過去3回の1日データを見て平均差枚・末尾を比較
  → 今日は旧イベ外なら期待値を下げる
```

### アビバ綱島南（分析例・提案デフォルトではない）

> 台番・末尾の読み方の**参考例**。店舗提案時は `knowledge/stores/README.md` のよく行く店を優先すること。

| 日付系 | 公約（みんパチ・口コミ） | データでの確認ポイント |
|--------|--------------------------|------------------------|
| 1・11・21日 | 旧イベ、全店9時開店 | 総差枚、バラエティ散らし |
| 4のつく日 | 新ジャグラー推し | ジャグラー列の稼働・差枚 |
| 5のつく日 | 旧ジャグラー推し | 同上 |
| 7/7・11/11 | 混雑・抽選時間変更 | 末尾7・ゾロ目の実績（7/7データ） |
| 8/9 | 周年（年イチ） | 過去周年の機種・台番 |

## 3. 取材イベントの読み方

みんレポ・rx7038 に載る取材名の例:

- クロロ調査隊 / クロロエンジェル
- スロッター✕スロッター / スロッター✕ワールド
- 一撃「姫撃」など

### 取材日に見ること

1. **取材対象機種**（告知・スケジュール）
2. 結果レポートの**平均差枚・勝率・出率**
3. **並び**だったか**単品**だったか
4. 負け台も**高G**なら設定残りの可能性（後ヅモ）

## 4. 結果から傾向を昇格させる基準

| レベル | 条件 | 記録場所 |
|--------|------|----------|
| 確定 | 公式告知と当日データが一致 | events + stores |
| 有力 | 同条件で3回以上、同方向の結果 | stores「傾向」 |
| 可能性 | 1〜2回のみ、または他店と混同 | stores「仮説」 |
| 却下 | 告知と逆、または再現せず | 記録のみ（次回参考） |

## 5. 翌日レビュー（当日予想の答え合わせ）

当日予想（`knowledge/events/YYYY-MM-DD-*.md`）を作った日は、**翌日以降に必ず答え合わせ**をする。
予想を出しっぱなしにしない。X情報を強く反映する方針は、このレビューで初めて検証可能になる。

### 手順

1. みんレポ等で対象日の **総差枚・機種別・末尾別** を確認
2. 当日予想ファイルの末尾に「結果・的中判定」セクションを追記（テンプレは `knowledge/events/_template.md`）
3. 予想の各根拠（店舗公式 / X情報垢 / 旧イベ / 直近データ）ごとに **的中 / 外れ / 判定不能** を付ける
4. X情報垢の的中・外れは `playbooks/information-sources.md` の**的中率トラッキング表に1行追記**する
5. 2回以上再現した傾向は `knowledge/stores/` へ昇格（セクション4の基準に従う）

### 判定の目安

| 判定 | 目安 |
|------|------|
| 的中 | 推し店・推し機種・末尾が総差枚/機種別データで明確にプラス |
| 外れ | 推し対象がマイナス、または告知内容が実施されなかった |
| 判定不能 | データ未公開・稼働薄で判断材料なし（トラッキングには数えない） |

### トラッキングの使い方（減点ルール）

- 直近10件で的中率が**3割を下回った垢**は、扱いを「有力」→「可能性」に格下げして予想への重みを落とす
- 逆に的中が続く垢は主根拠として維持。判断に迷ったら件数を貯めてから動かす
