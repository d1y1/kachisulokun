# 店舗・イベント偵察プレイブック

## 参照プレイブック

| ファイル | 内容 |
|----------|------|
| `playbooks/information-sources.md` | 情報源一覧・優先順位 |
| `playbooks/x-event-analysis.md` | X告知と過去イベント結果の突合 |
| `playbooks/seat-number-analysis.md` | 台番・末尾の癖考察 |
| `knowledge/stores/<店舗>.md` | 店舗別の蓄積ナレッジ |

## 店舗を見る観点

- イベント種別（設定公開、○台以上、○○の日 など）
- **X / LINE の当日・前日告知**
- 機種入れ替え・新台状況
- 過去の「当たりやすい日」仮説（曜日・旧イベ・ゾロ目）
- **末尾・並び・台番レンジの癖**（みんレポ・モグ玄）
- 閉店間際や特定時間帯の傾向（分かる範囲）

## イベント整理

1. 日付と内容を `knowledge/events/` に残す
2. 「公式に確かなこと」と「自分の仮説」を分ける
3. **告知内容と翌日の差枚データを突合**する
4. 当日の狙い機種・回避機種・**注目末尾**を短く書く

## 出力の心がけ

iOSチャット向けに、長文の店舗論ではなく:

- 今日の仮説 3行以内
- 見る台の優先順（機種・末尾・並び）
- 撤退ライン
- 情報の確度（確定/有力/可能性）
